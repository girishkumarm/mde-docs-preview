<Info>
**Source of truth**: the `market-data-engine` monorepo on `main`. Updated after a full code-audit of `services/`, `shared/`, `docker-compose.yml`, and `settings.gradle`. This page is the authoritative answer for *"what services exist and how do I run them?"*
</Info>

## At a glance

- **14 Spring Boot services** in the monorepo (13 runtime + 1 dev mock)
- **2 shared modules**: `shared/common` (enums, value types, security stubs), `shared/proto` (Protobuf contracts for Kafka + WebSocket)
- **Runtime stack**: Java 21 + Spring Boot 3.2.11 + Kafka 3.8.0 + ClickHouse JDBC 0.6.5 + Lettuce 6.4.0 + Temporal 1.26.2
- **Build**: Gradle multi-module, Spotless + Palantir Java Format enforced, Foojay auto-JDK
- **All services** expose OpenAPI 3.0 via springdoc-openapi at `/v3/api-docs` + `/swagger-ui.html`
- **All services** expose Prometheus metrics at `/actuator/prometheus`, health at `/actuator/health`

</Note>

## Service details

### instrument-master {#instrument-master}

**Purpose**: Scrip/instrument master CRUD + search + corporate-action sync.
**Port**: 8081 (mgmt 9081). **Class**: `InstrumentMasterApplication`.

| Endpoint | Method | Notes |
| --- | --- | --- |
| `/v1/instruments` | GET | List/filter (symbol, exchange, segment, scripType); paginated |
| `/v1/instruments/{instrumentToken}` | GET | Single instrument |
| `/v1/companies` | GET/POST | Company info |
| `/v1/search` | GET | Full-text search |
| `/internal/ca-sync` | GET/POST | **Unauthenticated** corporate-action webhook — called by publisher |
| `/internal/admin/*` | | **Unauthenticated** operator admin |

**Stores**: PostgreSQL (canonical), Redis Feed (hot lookups), S3 (scrip master snapshots).

### feed-handler {#feed-handler}

**Purpose**: Receives NSE + BSE multicast UDP, decodes, cross-source de-dups, publishes to Kafka.
**Port**: 8082. **Class**: `FeedHandlerApplication`.

- Consumes 6 UDP streams: `{NSE,BSE} × {CASH, FNO_0, FNO_1}` → 2 × 3 = 6 streams (docker-compose env).
- Produces ticks on `pml-marketfeed-{nse|bse}-{equity|derivative|index}-ticks` (see [Kafka topics](/data-apis/kafka-topics)).
- Also consumes `pml-marketfeed-{nse|bse}-derivative-ticks` internally for cross-source dedup.
- Admin endpoints under `/internal/admin/*`.

### publisher {#publisher}

**Purpose**: Consume ticks → write hot snapshots to Redis Feed (`FEED_PP_{segment}:{pmlId}`, `CANDLE:{interval}:{token}`, movers zsets) + Lua atomic 52-week H/L + OHLC compare-and-set.
**Port**: 8083. **Class**: `PublisherApplication`.

### market-movers-writer {#market-movers-writer}

**Purpose**: Rank top N by price change / volume / value across segments.
**Port**: 8084. **Class**: `MarketMoversWriterApplication`. **REST**: `GET /v1/movers`.

### fno-data-writer {#fno-data-writer}

**Purpose**: Option-chain aggregation, Black-Scholes Greeks, PCR, Max Pain, ATM Straddle, CPR.
**Port**: 8085. **Class**: `FnoDataWriterApplication`.

| Endpoint | Method | Notes |
| --- | --- | --- |
| `/v1/fno-chain` | GET/POST | Option chain aggregated data |
| `/v1/oi-buildup` | GET | Open-interest buildup analytics |
| `/v1/expiry-calendar` | GET | Expiry dates by underlying |

### candle-writer ⭐ {#candle-writer}

**Purpose**: The current (v2) dual-sink candle builder. Replaces `charts-writer` + `charts-historical-writer`.
**Port**: 8086. **Class**: `CandleWriterApplication`.

- Writes running candles (5s / 15s / 30s / 1m) to Redis Feed (`CANDLE:RUN:{interval}:{token}`, 120s TTL).
- Writes closed bars (1m+) to ClickHouse `candles_1m` via batched INSERT; publishes to `mde-candles-1m` Kafka topic for downstream consumers.
- Superseded commits: `22b566c` (merge), `35560c0` (schema), `bb3f8ff` (API switch), `a9863de` (Redis auto-config fix).

### charts-writer {#charts-writer}

<Warning>
**Deprecated** — superseded by `candle-writer`. Still in `settings.gradle` for backwards-compat but new deployments should not include it. Tracked for removal after v2 bake-off passes.
</Warning>

### broadcast-service {#broadcast-service}

**Purpose**: WebSocket fan-out of live ticks to authenticated sessions. Per-pod Kafka consumer group (unique group id per instance) — see [Kafka vs Redis ADR](/decisions/kafka-vs-redis).
**Port**: 8087. **Class**: `BroadcastServiceApplication`.

| Endpoint | Method | Notes |
| --- | --- | --- |
| `/ws/feed` | WebSocket | Live tick stream (Protobuf wire format) |
| `/v3/api-docs`, `/swagger-ui.html` | GET | OpenAPI (REST-only — WS not documented there) |

### recon-service {#recon-service}

**Purpose**: End-of-day OHLCV reconciliation against CMOTS + exchange reference; Temporal workflow. Publishes `mde-recon-reports`.
**Port**: 8089. **Class**: `ReconServiceApplication`. **REST**: `GET /v1/recon`.

### charts-historical-writer {#charts-historical-writer}

**Purpose**: Batched ClickHouse insert of `mde-candles-1m` — Kafka-only service, no REST.
**Port**: 8090. **Class**: `ChartsHistoricalWriterApplication`.

<Note>
After candle-writer went live (commit `22b566c`), this service is often rolled into the candle-writer deployment. Kept separate in code for backfill/batch scenarios.
</Note>

### premarket-scheduler {#premarket-scheduler}

**Purpose**: Temporal workflow coordinator — triggers pre-market jobs (scrip refresh, warm-up ticks, cache prime).
**Port**: 8086. **Class**: `PremarketSchedulerApplication`.

### incident-manager {#incident-manager}

**Purpose**: AlertManager webhook receiver → creates Jira tickets + Slack notifications + persists incident timeline.
**Port**: 8092. **Class**: `IncidentManagerApplication`.

| Endpoint | Method | Notes |
| --- | --- | --- |
| `/v1/incidents` | GET | List/filter/sort (paginated) |
| `/v1/incidents/{id}` | GET | Single incident |
| `/v1/incidents/{id}/acknowledge` | POST | On-call ack |
| `/internal/alert-webhook` | POST | **Unauthenticated** AlertManager receiver |

Ties into Decision 11 — AI-Powered Incident Manager from Day 1.

### jira-mock {#jira-mock}

**Purpose**: Local Atlassian API emulator for developing incident-manager without hitting real Jira.
**Port**: 8095. **Class**: `JiraMockApplication`. **Dev only** — never deployed.

### charts-api {#charts-api}

**Purpose**: Public read API for charts.
**Port**: 8086. **Class**: `ChartsApiApplication`.

| Endpoint | Method | Notes |
| --- | --- | --- |
| `GET /v1/candles/{instrumentToken}` | GET | OHLCV range. Query: `from`, `to`, `interval`, `source={redis\|clickhouse\|auto}`. `source=auto` (default) picks Redis for current day, ClickHouse for older. |
| `GET /v1/running-candles/{token}` | GET | In-progress candle (Redis only). Query: `interval` (default 5s). |
| `POST /v1/price-charts` | POST | TradingView UDF-compatible — keeps the PML UI working with zero config change (commit `bb3f8ff`). |
| `GET /v1/index` | GET | Index quotes (NIFTY 50, SENSEX, etc.) |
| `GET /v3/api-docs`, `/swagger-ui.html` | GET | OpenAPI |

**Caches**: Caffeine L1 per pod + Redis API cache-aside (`candle:1m:{token}`, 60s intraday / 30d historical).

**Redis auto-config**: disabled (commit `a9863de`) because two qualified `RedisConnectionFactory` beans (Feed + API) collide with Spring Boot defaults.

## Deployment groups

The 13 runtime services roll up into **5 deployment groups** (architecture Decision 6 — Service Decommission):

<CardGroup cols={2}>
  <Card title="Ingest group">
    `feed-handler` + `publisher` (latency-critical, co-located)
  </Card>
  <Card title="Writer group">
    `candle-writer` (+ `charts-historical-writer` for backfill), `fno-data-writer`, `market-movers-writer`
  </Card>
  <Card title="API group">
    `charts-api`, `instrument-master`, `market-movers-api`, `fno-api` (scale on req/s)
  </Card>
  <Card title="Broadcast group">
    `broadcast-service` (WebSocket-heavy, sticky-session LB)
  </Card>
  <Card title="Platform group">
    `premarket-scheduler` (Temporal), `recon-service` (Temporal), `incident-manager`
  </Card>
</CardGroup>
