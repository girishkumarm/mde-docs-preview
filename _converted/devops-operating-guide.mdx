## 3. Database strategies

### 3.1 PostgreSQL (RDS)

- Databases: `mde`, `incident`, `temporal` — one role per DB.
- Flyway migrations run in-service on boot.
- Backups: automated daily snapshots, 7-day retention.
- Max connections: 100.

### 3.2 Kafka

- Topics to pre-create (retention `7 200 000 ms` = 2 h for tick topics; `604 800 000 ms` = 7 d for event topics):

| Topic | Partitions | RF | cleanup |
| --- | --- | --- | --- |
| `pml-marketfeed-nse-equity-ticks` | 6 | 1 | delete |
| `pml-marketfeed-nse-fno-ticks` | 12 | 1 | delete |
| `pml-marketfeed-nse-index-ticks` | 3 | 1 | delete |
| `pml-marketfeed-bse-equity-ticks` | 3 | 1 | delete |
| `pml-marketfeed-bse-fno-ticks` | 3 | 1 | delete |
| `pml-marketfeed-bse-index-ticks` | 3 | 1 | delete |
| `mde-instrument-master-events` | 3 | 1 | compact |
| `mde-corporate-actions` | 3 | 1 | compact |
| `mde-recon-reports` | 3 | 1 | delete |
| `mde-incident-alerts` | 3 | 1 | compact |

### 3.3 ClickHouse

- Database: `mde`.
- Engines: `MergeTree` (single-replica) for `candles_1m`; `AggregatingMergeTree` for `candles_5m/15m/1h/1d`. No `ReplicatedMergeTree`, no ZooKeeper.
- Materialized views auto-populate the rollup tables from `candles_1m`.
- TTL: 60 d for `candles_1m` and `candles_5m`, 180 d for `candles_15m`/`candles_1h`, 1 y for `candles_1d`.
- Daily `clickhouse-backup` → S3 `mde-staging-backups/clickhouse/` (14-d retention).

### 3.4 Redis Cluster 1 — Feed (cache.r7g.xlarge, 1 primary + 1 replica)

- `maxmemory-policy volatile-lru`, `appendonly yes`, `appendfsync everysec`.
- Key patterns:

| Pattern | TTL |
| --- | --- |
| `CANDLE:RUN:5s\\|15s\\|30s\\|1m:{token}` | 120 s |
| `CANDLE:1m:{token}:{epoch_min}` + index ZSET | 3 h |
| `CANDLE:5m:{token}:{epoch_5m}` + index | 6 h |
| `CANDLE:15m:...` + index | 1 d |
| `CANDLE:1h:...` + index | 7 d |
| `FEED_PP_NSE_E:{pmlId}` | 8 h |
| `MGR_NSE_E_OHLC\\|PC\\|LC_UC:{pmlId}` | 8 h |
| `MGR_NSE_E_52_WK_HL:{pmlId}` | 24 h |

- Daily snapshot to S3 (ElastiCache automatic, 7-d retention).

### 3.5 Redis Cluster 2 — API (cache.r7g.medium, single node)

- `maxmemory-policy noeviction` — writers trim ZSETs explicitly.
- Holds `MOVERS:MOST_ACTIVE`, `MOVERS:TOP_GAINERS`, `MOVERS:TOP_LOSERS` + charts-api response cache with 60-s TTL.

### 3.6 Temporal

- Schedules: `im-daily-sync`, `recon-post-market-daily`, `bod-preload-daily`.
- Persistence: `temporal` DB on the shared RDS instance.
- Namespace: `default`.

## 5. Networking

- Single AZ (`ap-south-1a`).
- 1 NAT Gateway for egress.
- 1 internal ALB with NGINX ingress controller; `proxy-read-timeout: 3600s` for WebSocket upgrade.
- Route 53 private zone `mde-staging.internal.ap-south-1.staging.lending-v2.risk.pai.mypaytm.com`.
- Hosts:

- `mde-dashboard.<zone>` → instrument-master
- `mde-charts-api.<zone>` → charts-api
- `mde-broadcast.<zone>` → broadcast-service (WebSocket)
- `mde-grafana.<zone>` → Grafana
- `mde-temporal.<zone>` → Temporal UI
